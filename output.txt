🔧 How Slack OAuth works

Normal flow:

1. Click "Connect Slack"
2. Redirect to Slack
3. User logs in (OTP)
4. Slack redirects back to your app with:
   ?code=XYZ
5. Backend exchanges code → token
✅ What you do in tests

Skip steps 2–3 entirely

🟢 Test flow becomes:
1. Click "Connect Slack"
2. Intercept redirect
3. Inject fake OAuth response
4. Call your backend callback
5. Verify integration
🔥 Implementation (Playwright)
📄 Update your Slack test
test('Slack integration (mocked)', async ({ page }) => {
  await page.goto('https://merxus.ai/dashboard');

  await page.click('text=Settings');
  await page.click('text=Integrations');

  // Simulate OAuth callback instead of real login
  await page.goto(
    'https://merxus.ai/api/integrations/slack/oauth/callback?code=TEST_CODE'
  );

  // Verify connected
  await expect(
    page.locator('text=Slack connected')
  ).toBeVisible();
});
